# 1. 概述

**GhostEscape** 是一个基于 C++ 和 SDL3 构建的 2D 游戏引擎，包含：

- 层级实体系统（Object → ObjectScreen → ObjectWorld → Entity）
- 组件化架构（Sprite, SpriteAnim, Collider, States）
- 带相机支持的世界/场景管理
- 武器和法术系统（带冷却机制）
- 自动资源管理

```
│   │   └── Enemy (AI 控制 - 追踪玩家)
│   │
```

## 3.2 核心类详细说明

#### **Object** (基类)

```cpp
class Object {
protected:
    Game &m_game;                           // 单例引用
    std::vector<Object*> m_children;        // 子对象
    std::vector<Object*> m_children_back;   // 待添加的子对象 (安全添加)
    bool m_is_activive = true;             // 激活状态
    bool m_need_remove = false;            // 标记待移除
    ObjectType m_type = ObjectType::NONE;   // 类型分类

public:
    virtual void init();                    // 初始化 (不在构造函数中)
    virtual void handleEvents(SDL_Event&);
    virtual void update(const float& delta_time);
    virtual void render();
    virtual void clean();                   // 清理 (不在析构函数中)
    virtual void addChild(Object*);
    virtual void removeChild(Object*);
    void safeAddChild(Object*);             // 线程安全的子对象添加
};
```

**关键设计**：约定俗成 - 构造函数中不初始化，析构函数中不清理。

#### **ObjectScreen** (屏幕空间对象)

```cpp
class ObjectScreen : public Object {
protected:
    glm::vec2 m_render_position = glm::vec2(0);  // 屏幕位置
};
```

#### **ObjectWorld** (世界空间对象)

```cpp
class ObjectWorld : public ObjectScreen {
protected:
    glm::vec2 m_position = glm::vec2(0);    // 世界坐标 (绝对)
    Collider* m_collider = nullptr;        // 可选碰撞体
};
```

**关键特性**：坐标同步 - 设置世界坐标或渲染坐标会自动根据相机更新另一个。

#### **Entity** (游戏实体)

```cpp
class Entity : public ObjectWorld {
protected:
    States* m_states = nullptr;            // 生命/法力/伤害系统
    glm::vec2 m_velocity = glm::vec2(0);   // 当前速度
    float m_max_speed = 100.0f;            // 速度上限
};
```

## 4.3 SpriteAnim (动画精灵组件)

```cpp
class SpriteAnim : public Sprite {
private:
    int m_current_frame = 0;
    int m_total_frames = 0;
    int m_fps = 10;
    float m_frame_interval = 0.1f;
    float m_frame_timer = 0.0f;
```

**动画逻辑**:
- 每帧更新 `m_frame_timer`

# 8. 游戏循环

## 8.1 主循环结构

```cpp
void Game::run() {
    while (m_is_running) {
        auto start = SDL_GetTicksNS();

        handleEvents();           // 输入处理
        update(m_delta_time);     // 游戏逻辑
        render();                 // 渲染

        // 帧率控制 (目标 60 FPS)
        if (elapsed < m_frame_delay) {
            SDL_DelayNS(m_frame_delay - elapsed);
            m_delta_time = frame_delay / 1e9;  // 稳定 delta
        } else {
            m_delta_time = elapsed / 1e9;      // 实际耗时
        }
    }
}
```

## 8.2 帧更新顺序

```
1. Scene::update()
   ├── Object::update() (处理待添加的子对象)
   │   ├── 将 m_children_back 添加到 m_children
   │   └── 移除标记的子对象
   │
   ├── m_children_world 更新
   │   ├── Player::update() - 处理输入、移动、同步相机
   │   ├── Enemy::update() - 追踪玩家、攻击
   │   ├── Spell::update() - 检查碰撞
   │   ├── Effect::update() - 检查动画结束
   │   ├── Spawner::update() - 生成敌人
   │   ├── States::update() - 回复法力、更新无敌帧
   │   └── Weapon::update() - 更新冷却
   │
   └── m_children_screen 更新
       └── UIMouse::update() - 切换光标精灵

2. 从容器中移除标记的对象
```

## 8.3 渲染顺序

```
1. 清屏 (黑色)
2. SceneMain::render()
   ├── renderBackground() - 绘制网格和边界
   └── Scene::render()
       ├── 渲染 m_children_world (世界层)
       └── 渲染 m_children_screen (UI 层)
3. 显示渲染器
```

# 10. 设计模式

## 10.1 单例模式 (Game)

```cpp
class Game final {
private:
    Game() = default;
    Game(const Game&) = delete;
    Game& operator=(const Game&) = delete;

public:
    static Game& getInstance() {
        static Game instance;  // Meyers' Singleton
        return instance;
    }
};
```

## 10.2 组件模式

对象由可附加组件组成：
- **Entity** (Player, Enemy) = ObjectWorld + States + SpriteAnim + Collider
- **Spell** = ObjectWorld + SpriteAnim + Collider
- **Effect** = ObjectWorld + SpriteAnim

组件通过静态工厂方法创建：
```cpp
static SpriteAnim* addSpriteAnim(ObjectScreen*, file_path, scale, anchor);
static Collider* addCollider(ObjectScreen*, size, type, anchor);
static States* addStates(Entity*, max_health, max_mana, mana_regen, damage);
```

## 10.3 层级更新模式

```cpp
void Object::update(const float& delta_time) {
    // 处理待添加的子对象
    for (auto& child : m_children_back) {
        addChild(child);
    }
    m_children_back.clear();

    // 更新子对象 (安全移除)
    for (auto it = m_children.begin(); it != m_children.end();) {
        if ((*it)->getNeedRemove()) {
            (*it)->clean();
            delete *it;
            it = m_children.erase(it);
        } else {
            if ((*it)->getActiveState()) {
                (*it)->update(delta_time);
            }
            ++it;
        }
    }
}
```

## 10.4 安全添加模式

避免迭代时添加子对象导致迭代器失效：
```cpp
void Object::safeAddChild(Object* child) {
    m_children_back.push_back(child);  // 添加到待处理
}
```

## 10.5 状态机模式 (Enemy)

```cpp
enum class State { NORMAL, HURT, DEAD };

void Enemy::checkState() {
    if (m_states->getHealth() <= 0) changeState(State::DEAD);
    else if (m_states->getIsInvincible()) changeState(State::HURT);
    else changeState(State::NORMAL);
}
```

# 12. 关键设计决策

## 12.1 为什么不使用虚析构函数？

约定明确：
- **构造函数中不初始化** - 使用 `init()`
- **析构函数中不清理** - 使用 `clean()`

这允许：
- 初始化失败可以返回而不产生半构造对象
- 显式控制销毁顺序
- 调试友好的清理日志

## 12.2 为什么要两个子对象容器？

- `m_children` - 通用对象层级树
- `m_children_screen` - 屏幕固定元素（不随相机移动）
- `m_children_world` - 世界元素（随相机移动）

这种分离允许高效的渲染顺序和相机相关更新。

## 12.3 为什么要用 ObjectAffiliate 作为组件基类？

组件需要：
- 父对象引用用于位置计算
- 偏移/锚点系统用于定位
- 大小用于碰撞/渲染

ObjectAffiliate 提供这些共享能力而不重复代码。

## 12.4 为什么要用懒加载纹理？

游戏资源可能很大。懒加载：
- 减少启动时间
- 只加载实际使用的
- 后续访问命中缓存

# 总结

GhostEscape 引擎实现了一个结构良好的层级实体系统，包含：

1. **关注点清晰分离**: 对象层级负责结构，组件负责行为
2. **高效渲染**: 分层容器，相机相关定位
3. **生命周期管理**: init/clean 约定，安全添加模式
4. **灵活组合**: 组件可混搭
